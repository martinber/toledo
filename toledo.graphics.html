toledo.graphics.screen module¶

class toledo.graphics.screen.Screen(size)[source]¶

Bases: object

Se encarga de controlar lo que ve el jugador en la pantalla.

Primero se debe crear una instancia de este objeto, luego, en cada loop del juego uno debe (en orden):

  • Pintar la pantalla con fill() para tapar el frame anterior.
  • Dibujar todos los sprites con draw().
  • Actualizar la pantalla con update().

draw(sprite, rect, angle=0, smooth=False, anchor=None)[source]¶

Dibujar un sprite en la pantalla.

No se verán los cambios hasta que se llame a la función Screen.update().

Parameters:
  • sprite (toledo.graphics.Sprite) – Sprite a dibujar.
  • rect (toledo.Rect) – Posición en la pantalla en donde dibujar. Se respeta el tamaño dado.
  • angle (float, optional) – Ángulo de rotación en grados, sentido contrario a las agujas del reloj.
  • smooth (bool, optional) – Si usar o no antialiasing.
  • anchor (int, optional) – Tipo de anclaje a usar, (por ej. si centrado o esquinado). Usar las constantes definidas. Ej: Screen.ANCHOR_CENTER o Screen.ANCHOR_TOP_LEFT. Por defecto es un anclaje top-left.

Examples

screen = toledo.graphics.Screen(screen_size)
...
def loop():
    ...
    screen.draw(sprite_ball, rect_ball, angle=angle_ball,
                smooth=True, anchor=screen.ANCHOR_TOP_LEFT)
